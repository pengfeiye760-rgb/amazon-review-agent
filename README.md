# Amazon Review Analysis Agent

## Project Overview

This project is a prototype of an intelligent software agent for analysing Amazon product reviews.

The agent takes a review CSV file as input, observes the dataset structure, decides an analysis path, classifies review issues, computes category statistics, and recommends quality-control actions.

The goal is to help identify product defect patterns from customer reviews and support product improvement decisions.

---

## Agent Workflow

The system follows an agentic workflow:

1. **Perception / Observation**
   - Reads the uploaded CSV file.
   - Checks dataset columns, number of rows, missing comments, and low-rating ratio.

2. **Decision Making**
   - Decides whether the analysis should be balanced or defect-focused.
   - Detects missing fields and data quality issues.
   - Uses schema recovery when possible, such as using `case_id` as `review_id`.

3. **Action / Tool Use**
   - Calls deterministic Python tools to classify reviews.
   - Computes category statistics with pandas.
   - Generates charts and QC recommendations.

4. **Safety and Reliability**
   - Numerical statistics are computed by Python, not generated by an LLM.
   - Review labels are selected from a predefined defect taxonomy.
   - Unknown or vague reviews are marked as `unknown`.
   - Edge-case data triggers warnings when the result may require manual review.

## How to Run Locally

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Start the Streamlit app

```bash
streamlit run app.py
```

### 3. Upload a CSV file

Use one of the example datasets:

```text
data/sample_reviews.csv
data/edge_case_reviews.csv
```

Then click:

```text
Run Agent Analysis
```

The app will display:

- Dataset preview
- Agent observation summary
- Agent decision and warnings
- Review classification results
- Category statistics
- Bar chart
- Quality-control recommendations

---

## How to Run Evaluation

The project includes a labelled evaluation dataset:

```text
data/labelled_eval_reviews.csv
```

Run:

```bash
python evaluation.py
```

This compares the agent's predicted labels with human labels and saves detailed results to:

```text
data/evaluation_results.csv
```

The evaluation output includes:

- Total number of labelled examples
- Correct predictions
- Classification accuracy
- Incorrect cases for manual inspection

## Why This Is an Agent

This system is not only a fixed dashboard or traditional reporting script.

It follows an agent-style workflow:

- It **observes** the uploaded dataset.
- It **decides** whether the analysis should be balanced or defect-focused.
- It **recovers** from minor schema differences, such as using `case_id` when `review_id` is missing.
- It **acts** by calling classification, statistics, charting, and recommendation tools.
- It **warns** the user when the dataset has quality issues, such as many unknown classifications.
